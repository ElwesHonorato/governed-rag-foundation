# Repository Pattern Standards

This folder is the central place for architectural and coding patterns that should be enforced in this repository.

## How to use
1. Add one markdown file per enforceable pattern/rule.
2. Keep each file practical: intent, required rules, anti-patterns, and review checklist.
3. Link every new pattern file in this index.
4. Prefer explicit "must/must not" wording for enforceable rules.
